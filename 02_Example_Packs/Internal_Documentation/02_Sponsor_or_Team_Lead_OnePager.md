# Sponsor / Team Lead One-Pager

## Pilot summary
This pilot tests a simple method for using AI in low-risk internal documentation work. It is intentionally limited in scope. The purpose is to assess whether AI can be used in a controlled way without weakening ownership, review, or evidence.

## Pilot design
- Duration: 30 days
- Scope: one team or function
- Use case type: internal drafting and support work
- Decision principle: humans remain responsible
- Output principle: no release or operational use without review

## Main controls
- named task owner
- named reviewer
- named approver
- AI use log
- weekly scorecard
- retained review evidence
- clear exclusions for high-risk use

## Suggested success criteria
The pilot should be considered workable if the team can show:
- basic compliance with the workflow
- acceptable document quality after review
- manageable workload for reviewers
- no uncontrolled data use
- no confusion about ownership or approval
- clear evidence for at least a sample of pilot outputs

## Decision points for leadership
At the end of the pilot, leadership should decide whether to:
- stop
- extend
- scale in a limited form
- formalise a wider internal method

## Minimum leadership actions
- nominate admin / coordinator for the pilot
- confirm approver
- approve the pilot boundary
- request weekly reporting
- review the final evidence summary

# Generic Role Map

## Purpose

This document defines the standard roles used in Genesis Coherence. It is generic by design and is not tied to one sector or pilot.

## Standard roles

### Sponsor
The Sponsor authorizes the work, confirms the business or institutional need, and removes blockers when needed.

### Task Owner
The Task Owner is responsible for completing the task, keeping records current, and making sure work follows the agreed rules.

### Reviewer
The Reviewer checks the output for quality, completeness, and fitness for purpose before approval.

### Approver
The Approver gives the formal go-ahead for release, submission, publication, or operational use.

### Admin / Coordinator
The Admin or Coordinator maintains the workflow, tracks status, stores records, and ensures the right people are involved at the right points.

### Legal / Data Reviewer (optional)
This role is used only when the work involves legal, privacy, data protection, confidentiality, or policy risk that requires specialist review.

## Basic role rules

- Every governed task should have one Task Owner.
- Every governed task should have one Approver.
- Review and approval should not be merged unless the activity is low risk and this is explicitly accepted.
- The Sponsor does not need to be involved in every task.
- The Legal / Data Reviewer is used only when needed.
- The Admin / Coordinator may support many tasks but should not become the default decision-maker.

## Minimum assignment pattern

For most small workflows:
- Sponsor: optional at task level
- Task Owner: required
- Reviewer: recommended
- Approver: required
- Admin / Coordinator: recommended
- Legal / Data Reviewer: conditional
